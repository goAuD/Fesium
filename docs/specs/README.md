## Conventions

- One spec per dated feature. Filename: `YYYY-MM-DD-topic-design.md`.
- Specs are frozen historical records. If a design changes, capture the change in a newer spec, not by rewriting the old one.
- Each spec has a matching implementation plan under [../plans/](../plans/) with the same date prefix.
