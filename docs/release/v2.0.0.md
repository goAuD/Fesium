# Fesium v2.0.0 — 2026-04-19

The `NanoServer` project is now `Fesium`. This release is the structured migration to the new brand, package layout, and product shell. Backend behavior is intentionally preserved; the UI and repo contract are what changed.

## Headline Changes

- Rebranded from `NanoServer` to `Fesium` across the app, docs, and public metadata.
- New `src/fesium/` package layout replaces the old flat runtime modules.
- New desktop shell with a sidebar and dedicated views for Overview, Server, Database, Diagnostics, Guide, and Settings.
- Bundled offline fonts and the approved Graphite Grid visual direction.
- Restored and expanded local-server workflow: project selection, Laravel/standard detection, PHP serving, static fallback, open-in-browser.
- Restored SQLite workflow: project-detected or manually selected database, read-only default, single-statement SQL execution, destructive-query confirmation, focused schema browser with table list, column inspect, and a preview query.
- New student-facing **Guide** view.
- Replaced the old fake settings summary with an honest placeholder.
- Switched license from MIT to Apache-2.0.

## Migration Notes

- Both `python fesium.py` and `python nanoserver.py` work. `nanoserver.py` is a compatibility shim that will be removed in a future release.
- The legacy `~/.nanoserver/` config directory is still read as a fallback once, so existing users do not lose their preferences. New state is written to `~/.fesium/`.
- The old `requirements-dev.txt` is gone. Both runtime and contributor dependencies are in `requirements.txt`.

## Security

- SQLite read-only mode stays enabled by default and resets on every launch.
- Destructive queries (`DROP`, `DELETE`, `UPDATE`, `TRUNCATE`, `ALTER`) require explicit confirmation in write mode.
- Document-root existence is validated before server startup.
- GitHub Actions `python-tests` workflow hardened with explicit `contents: read` permissions.

## Breaking Changes

- The legacy root-level runtime modules (`server.py`, `database.py`, `config.py`, etc. at repo root) are removed.
- The old `test_nanoserver.py` suite is replaced by the modular `tests/` tree.
- `DESIGN_SYSTEM.md` and legacy root brand images (`nanoserver.png`, `social_preview.png`) are removed.
